Type & Shape

body uses Iowan Old Style, Palatino, Georgia serif. Extension pages now use the same brand type stack.

h1-h3 use letter-spacing: -0.02em. Buttons use compact control sizing; cards use 18px; chips and badges use the pill radius.

Common spacing: page gutter 18px, form/card gaps 18px, mobile gutter 8px.

Font Size Scale

The Browser Reader May 10, 2026 · 6 min read

How small design systems stay coherent

A representative article view showing headline scale, metadata, long-form body text, inline links, block quotes, and a compact contextual panel.

Consistency starts with fewer choices. A product does not need dozens of text sizes or button shapes to feel polished; it needs a small set of decisions that repeat across screens.

In EchoTree, article text is intentionally larger than interface text. Controls stay compact, while the reading surface gets more breathing room and a longer line height.

Use the scale by role

The smallest size belongs to metadata and statuses. The small size belongs to controls. The medium size supports page structure. The large size carries sustained reading.

When every component invents its own size, the interface starts to feel assembled instead of designed.

That separation lets dense operational screens and relaxed reading screens coexist without fighting the same visual hierarchy.

Consistency Notes

Typography follows a compact scale: 9px for metadata/status, 11px for controls, 13px for supporting UI copy, and 15-18px for article text.

Button styling is consistent and compact: native buttons and .button links share padding, radius, line height, and hover states.

The extension UI follows the app palette and type. Its tokens are duplicated locally because extension pages are standalone documents.
